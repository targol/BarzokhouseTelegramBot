# ربات رزرو اقامتگاه بومگردی «خانه برزک»

## ساختار پروژه

```
telegram-booking-bot/
├── bot.py               ← منطق اصلی ربات (نیازی به تغییر نیست)
├── config.py             ← تمام متن‌ها، اتاق‌ها، قیمت‌ها و اطلاعات تماس (اینجا رو ویرایش کن)
├── requirements.txt       ← کتابخانه مورد نیاز
├── rooms_photos/          ← عکس اتاق‌ها رو اینجا بذار
└── README.md
```

## مراحل راه‌اندازی

### ۱. ساخت ربات در تلگرام
۱. به [@BotFather](https://t.me/BotFather) پیام بده.
۲. دستور `/newbot` رو بزن و اسم و یوزرنیم دلخواه رو وارد کن.
۳. توکنی که بهت میده رو کپی کن.

### ۲. گرفتن آیدی عددی برای دریافت درخواست‌های رزرو
۱. به [@userinfobot](https://t.me/userinfobot) پیام بده تا آیدی عددی خودت رو بگیری.
۲. اگر می‌خوای درخواست‌ها به یک گروه ارسال بشه، ربات رو به اون گروه اضافه کن و آیدی عددی گروه رو (که با علامت منفی شروع میشه) بگیر.

### ۳. تکمیل فایل config.py
فایل `config.py` رو باز کن و موارد زیر رو با اطلاعات واقعی جایگزین کن:
- `BOT_TOKEN` → توکنی که از BotFather گرفتی
- `ADMIN_CHAT_ID` → آیدی عددی که در مرحله قبل گرفتی
- `WELCOME_MESSAGE`, `ADDRESS_TEXT`, `CONTACT_TEXT`, `SOCIAL_TEXT` → متن‌های دلخواه
- `LOCATION_LATITUDE` و `LOCATION_LONGITUDE` → مختصات دقیق اقامتگاه (می‌تونی از گوگل مپ بگیری: راست‌کلیک روی نقطه → کپی مختصات)
- `ROOMS` → اطلاعات هر اتاق (نام، توضیحات، قیمت، مسیر عکس)

### ۴. اضافه کردن عکس اتاق‌ها
عکس هر اتاق رو با همون اسمی که در `config.py` مشخص کردی (مثلاً `shatoot.jpg`) داخل پوشه `rooms_photos/` بذار.

### ۵. نصب و اجرا
```bash
pip install -r requirements.txt
python bot.py
```

اگر همه چیز درست باشه، در ترمینال پیام «ربات در حال اجراست...» رو می‌بینی. حالا می‌تونی در تلگرام دستور `/start` رو به ربات بزنی.

## نحوه کارکرد ربات

1. کاربر `/start` می‌زنه → پیام خوش‌آمدگویی + منوی اصلی (آدرس، تماس با ما، شبکه‌های اجتماعی، اتاق‌ها) نمایش داده میشه.
2. با زدن «اتاق‌ها» لیست ۵ اتاق (شاتوت، قالی، آبی، سرا، بالاخونه) نشون داده میشه.
3. با انتخاب هر اتاق، عکس + توضیحات + قیمت اون اتاق به همراه دکمه «درخواست رزرو» نمایش داده میشه.
4. با زدن «درخواست رزرو»، ربات از کاربر نام و سپس شماره تماس رو می‌پرسه.
5. بعد از تکمیل اطلاعات، یک پیام شامل نام اتاق، قیمت، نام کاربر، شماره تماس و آیدی تلگرامش به `ADMIN_CHAT_ID` ارسال میشه و به کاربر هم پیام تایید نشون داده میشه.

## اجرای رایگان و ۲۴ ساعته روی سرور (Render)

چون تلگرام یک سرور همیشه‌روشن (Always-on) لازم داره، ساده‌ترین گزینه‌ی **کاملاً رایگان و بدون کارت بانکی** در حال حاضر **Render.com** هست. کد این ربات از قبل برای این حالت آماده‌ست (به‌صورت خودکار وارد حالت webhook میشه).

⚠️ نکته مهم درباره پلن رایگان Render: سرویس رایگان بعد از ۱۵ دقیقه بدون درخواست، «می‌خوابه» و با اولین پیام کاربر (که تلگرام برای ربات ارسال می‌کنه) دوباره تا حدود ۱ دقیقه بعد بیدار میشه. یعنی اگر مدتی هیچ‌کس با ربات کار نکنه، اولین نفری که بعدش بیاد یکی-دو دقیقه معطل جواب اول می‌مونه؛ بعدش سرعت عادیه. برای یه اقامتگاه کوچیک با ترافیک کم این کاملاً قابل‌قبوله.

### مراحل دیپلوی روی Render

1. **ساخت ریپازیتوری گیت‌هاب**
   یک ریپازیتوری (می‌تونه Private هم باشه) بساز و همه فایل‌های این پروژه (`bot.py`, `config.py`, `requirements.txt`, `render.yaml`, پوشه `rooms_photos` با عکس‌های واقعی) رو داخلش push کن.
   ⚠️ چون توکن ربات و آیدی مدیر الان از متغیر محیطی خونده میشن (نه مستقیم از کد)، لازم نیست نگران لو رفتنشون در گیت‌هاب باشی — فقط مطمئن شو مقدار `BOT_TOKEN` و `ADMIN_CHAT_ID` رو داخل `config.py` دستکاری نکردی (پیش‌فرض بمونن).

2. **ساخت اکانت رایگان Render**
   به [render.com](https://render.com) برو و با گیت‌هاب اکانت بساز (نیازی به کارت بانکی نیست).

3. **ساخت وب سرویس جدید**
   - از داشبورد Render، گزینه **New → Web Service** رو بزن.
   - ریپازیتوری گیت‌هابی که ساختی رو انتخاب کن.
   - Render فایل `render.yaml` رو خودش تشخیص میده و تنظیمات (Build/Start Command و پلن Free) رو خودکار پر می‌کنه. اگه پرسید، پلن **Free** رو انتخاب کن.

4. **اضافه کردن متغیرهای محیطی (Environment Variables)**
   در بخش Environment سرویس، این دو مقدار رو اضافه کن:
   - `BOT_TOKEN` → توکنی که از BotFather گرفتی
   - `ADMIN_CHAT_ID` → آیدی عددی خودت (یا گروه)

5. **دیپلوی**
   دکمه Create/Deploy رو بزن. بعد از چند دقیقه، سرویس بالا میاد و یک آدرس مثل `https://barzok-booking-bot.onrender.com` بهت میده. کد ربات خودش این آدرس رو (از متغیر `RENDER_EXTERNAL_URL` که Render خودکار می‌سازه) می‌گیره و webhook تلگرام رو باهاش تنظیم می‌کنه — کار دیگه‌ای لازم نیست.

6. **تست**
   توی تلگرام به ربات `/start` بزن. اگه چند ثانیه تا ۱ دقیقه (به خاطر بیدار شدن سرویس) طول کشید، طبیعیه.

### اگه بعداً خواستی روی سیستم خودت هم تستش کنی
کافیه بدون تنظیم متغیر `WEBHOOK_URL` اجراش کنی؛ خودکار به حالت **polling** برمی‌گرده (همون روشی که قبلاً توضیح داده شد).

## نکات مهم برای استفاده واقعی (Production)

- در حال حاضر ربات فقط درخواست رزرو رو ثبت و ارسال می‌کنه و رزرو رو به‌صورت خودکار نهایی نمی‌کنه (نیاز به تایید دستی مدیر داره). اگر بخوای تقویم رزرو، جلوگیری از رزرو دوبل، یا پرداخت آنلاین هم اضافه بشه، باید یک پایگاه داده و منطق بیشتری اضافه کرد که خوشحال میشم در ادامه کمک کنم.
- روی پلن رایگان Render، فضای فایل موقت (ephemeral) هست، یعنی هر تغییری که هنگام اجرا روی فایل‌ها انجام بشه، بعد از هر ری‌استارت/دیپلوی از بین میره. چون این ربات چیزی روی دیسک ذخیره نمی‌کنه (فقط پیام می‌فرسته)، مشکلی نداره؛ ولی اگه بعداً دیتابیس اضافه کنیم باید از سرویس دیتابیس Render (یا مشابه) استفاده کنیم.
- شماره‌های تماس کاربران اطلاعات حساس هستن؛ اگر قراره این ربات رو عمومی منتشر کنی، به قوانین حریم خصوصی توجه کن.
- سایتی که الان روی GitHub Pages داری (`targol.github.io/...`) فقط می‌تونه فایل‌های استاتیک (HTML/CSS/JS) رو نمایش بده و امکان اجرای کد پایتون یا نگه‌داشتن یک پروسه‌ی همیشه‌روشن رو نداره؛ برای همین این ربات نمی‌تونه اونجا میزبانی بشه و باید از یه سرویس مثل Render استفاده کنی.
